# GitQuest 70-Minute Teaching Narrative

## Outcome
By the end, learners can explain and perform the full Git loop:
working directory -> staging area -> commit history -> branching -> collaboration -> recovery.

## Structure (70 minutes)
1. 0-5 min: Opening story + interface orientation
2. 5-20 min: Basics arc (Levels 1-6)
3. 20-40 min: Branching arc (Levels 7-12)
4. 40-55 min: Collaboration arc (Levels 13-18)
5. 55-65 min: Advanced safety arc (Levels 19-21)
6. 65-70 min: Sandbox challenge + recap (Level 22)

## Narrative Arc
- Act 1 (Basics): "You are bootstrapping a project and teaching Git what matters."
- Act 2 (Branching): "Your team splits work safely with parallel timelines."
- Act 3 (Collaboration): "You synchronize with remotes and resolve shared history."
- Act 4 (Advanced): "You recover from mistakes and shape clean history."

## Interaction Pattern For Every Level (2-3 minutes each)
1. Prediction: Ask learners to predict command output before running.
2. Execution: Run command(s) in terminal.
3. Interpretation: Read three areas: repo-state strip, graph, and workspace inspector.
4. Reflection: Ask one "why" question.

## High-Impact Interaction Prompts
- "Which files are untracked vs unstaged vs staged right now?"
- "What moved after this command: file content, staging area, or commit pointer?"
- "What do you expect the graph to do next?"
- "Should this be a merge or a rebase in a team setting?"

## Suggested Live Challenges
1. Basics challenge (after Level 6): Intentionally modify and stage only one of two files.
2. Branching challenge (after Level 10): Create divergence then predict fast-forward vs merge commit.
3. Collaboration challenge (after Level 17): Simulate remote ahead, then recover cleanly with pull/fetch.
4. Advanced challenge (after Level 21): Use reset modes and explain what each preserves.

## Instructor Timing Guide
- Keep command execution short; spend most time on prediction and interpretation.
- If stuck, click command coach chips to continue flow.
- Use graph replay controls (Prev/Next/Live) to revisit turning points.

## Closing (last 5 minutes)
1. Ask each learner to explain the 3 Git zones in one sentence each.
2. Run a final mini-scenario: create file -> add -> commit -> branch -> merge.
3. Assign follow-up: reproduce the scenario from memory without hints.
